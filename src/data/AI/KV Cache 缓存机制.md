# KV Cache 缓存机制

## 问题 1：KV Cache 缓存的是什么？

### 【问题】
KV Cache 缓存的是什么？是 QK 或 QKV 结果吗？

### 【回答】
KV Cache 缓存的是 K 和 V 矩阵，而非 QK 或 QKV 结果。

**原因**：
1. K/V 是历史 token 的固定投影，可被后续 Q 重复使用。
2. QK/QKV 依赖当前 Q，无法复用。

**优化意义**：
- 将自回归推理的复杂度从 O(n² · d_model) 降至 O(n · d_model)。
- MQA/GQA 通过共享 K/V 进一步减少显存占用（降至 1/h）。

### 问题难点分析
这个问题考察对 Transformer 推理优化的核心机制理解。难点在于理解为什么只缓存 K/V 而不缓存 Q，以及这种设计对计算复杂度的影响。需要理解自回归生成过程中历史 token 的复用特性。

## 问题 2：为什么 KV Cache 只缓存 K 和 V，而不缓存 Q？

### 【问题】
为什么 KV Cache 只缓存 K 和 V，而不缓存 Q？

### 【回答】
**核心原因**：K 和 V 是历史 token 的固定投影，而 Q 是随当前生成位置变化的。

**详细解释**：
- **K/V 的固定性**：在自回归生成中，已生成的历史 token 的 Key 和 Value 向量是固定的，不会随后续生成而改变。因此可以缓存起来供后续所有位置重复使用。
- **Q 的变化性**：Query 向量是针对当前要生成的 token 计算的，每个生成步骤的 Q 都不同，无法复用。
- **QK/QKV 的依赖性**：QK 点积和最终的注意力输出都依赖于当前的 Q，因此即使缓存了这些中间结果也无法复用。

**类比理解**：
可以把 K/V 想象成"历史档案"，一旦形成就不会改变；而 Q 是"当前问题"，每次提问都不同，需要根据当前情境实时生成。

### 问题难点分析
这个问题深入考察对 Attention 机制中 Q、K、V 角色差异的理解。难点在于理解自回归生成过程中，历史 token 和当前 token 的不同处理方式，以及这种不对称性带来的优化机会。

- **对自回归生成过程的理解**，能否说明生成过程中 token 的依赖关系。
- **工程优化思维**，是否能从计算特性中发现优化机会。

## 问题 3：KV Cache 如何将推理复杂度从 O(n²) 降至 O(n)？

### 【问题】
KV Cache 如何将自回归推理的复杂度从 O(n² · d_model) 降至 O(n · d_model)？

### 【回答】
**原始复杂度（无 KV Cache）**：
每次生成新 token 时，需要重新计算所有历史 token 的 K 和 V，然后与当前 Q 计算注意力。
- 第 i 个 token 需要计算 i 个 K/V
- 总计算量：1 + 2 + 3 + ... + n = O(n²)

**优化后复杂度（有 KV Cache）**：
- 缓存历史 token 的 K/V，只需计算当前 token 的 K/V
- 每个 token 的计算量变为常数级
- 总计算量：n × O(1) = O(n)

**具体计算**：
- 无缓存：∑(i=1 to n) i × d_model = O(n² · d_model)
- 有缓存：n × d_model = O(n · d_model)

**收益**：
对于长序列（如 n=4096），计算量从约 1600 万次操作降至约 4000 次操作，提升约 4000 倍。

### 问题难点分析
这个问题考察对算法复杂度分析和 KV Cache 实际效果的理解。难点在于理解自回归生成中重复计算的来源，以及缓存如何消除这种重复。

## 问题 4：MQA 和 GQA 如何进一步优化 KV Cache？

### 【问题】
MQA（Multi-Query Attention）和 GQA（Grouped-Query Attention）如何进一步优化 KV Cache 的显存占用？

### 【回答】
**标准 Multi-Head Attention（MHA）**：
- 每个头都有独立的 K 和 V
- KV Cache 大小：2 × n × h × d_head（h 为头数）

**MQA（Multi-Query Attention）**：
- 所有头共享同一组 K 和 V
- KV Cache 大小：2 × n × 1 × d_head = 2 × n × d_head
- 显存降至 1/h

**GQA（Grouped-Query Attention）**：
- 将头分成 g 组，每组共享 K 和 V
- KV Cache 大小：2 × n × g × d_head
- 显存降至 g/h（介于 MHA 和 MQA 之间）

**效果对比**（假设 h=8）：
| 方法 | KV Cache 大小 | 相对显存 |
|------|--------------|---------|
| MHA | 2 × n × 8 × d | 100% |
| GQA(g=2) | 2 × n × 2 × d | 25% |
| MQA | 2 × n × 1 × d | 12.5% |

**权衡**：
- MQA 显存最优但可能损失模型能力
- GQA 在显存和性能之间取得平衡
- LLaMA-2 等模型采用 GQA 作为折中方案

### 问题难点分析
这个问题考察对 KV Cache 进阶优化技术的理解。难点在于理解 MQA 和 GQA 的共享机制，以及这种共享带来的显存节省与潜在性能损失之间的权衡。
