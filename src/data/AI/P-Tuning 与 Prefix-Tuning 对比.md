# P-Tuning 与 Prefix-Tuning 对比

## 问题 1：P-Tuning（P-tuning v1）的主要特点

### 【问题】
P-Tuning（P-tuning v1）的主要特点

### 【回答】

**注入位置**：主要在输入层（Embedding Layer）注入可学习的连续前缀（Soft Prompts）。

**具体作用**：这些 Soft Prompts 直接拼接在原始输入文本的词嵌入前面，作为模型接收的初始信息。它们就像额外的"虚拟词"，引导模型从一开始就进入特定的任务模式。

**局限性**：由于只在输入层生效，这些提示对 Transformer 模型深层的影响可能会减弱，尤其对于需要复杂推理或多层信息交互的任务，效果可能有限。

**典型应用**：更侧重于自然语言理解（NLU）任务。

### 问题难点分析
这个问题的难点在于理解 P-Tuning 只在输入层添加可学习参数，这种浅层注入方式对深层网络的影响有限。需要理解为什么仅在 embedding 层添加前缀会导致对深层控制力不足。

### 面试官考察点
- **对 Prompt Tuning 技术演进的理解**，是否清楚 P-Tuning 的定位。
- **对模型架构的深刻理解**，能否分析出浅层注入的局限性。
- **对任务类型的把握**，是否知道 NLU 和 NLG 任务对 Prompt 技术的不同需求。

### 问题衍生
- P-Tuning 为什么更适合 NLU 任务而不是 NLG 任务？
- 如果要改进 P-Tuning 的深层影响力，你会怎么设计？
- P-Tuning 和传统 Prompt Engineering 有什么区别？

## 问题 2：Prefix-Tuning 的主要特点

### 【问题】
Prefix-Tuning 的主要特点

### 【回答】

**注入位置**：在 Transformer 模型的每一层（Layer-wise）都注入可学习的连续前缀，并且这些前缀是直接添加到注意力机制的键（Key, K）和值（Value, V）向量序列中。

**具体作用**：它们直接修改了每一层注意力计算时可用的 KV Cache。这意味着模型在计算注意力时，不仅考虑原始输入的 KV 向量，还会考虑这些额外的、任务特定的前缀 KV 向量。这能更深层次地影响模型的内部表示和注意力分配，从而对整个信息流产生更强的控制。

**典型应用**：最初主要为**自然语言生成（NLG）**任务设计，因为它能有效引导生成过程。

### 问题难点分析
这个问题的难点在于理解 Prefix-Tuning 是逐层注入的，直接影响每一层的 KV Cache。需要理解这种深层注入方式如何比 P-Tuning 更有效地控制模型行为，尤其是在生成任务中。

### 面试官考察点
- **对 Transformer 内部机制的理解深度**，是否清楚 KV Cache 的作用。
- **对 Prefix-Tuning 设计动机的理解**，为什么选择注入到 K 和 V 而不是 Q。
- **对 NLG 任务特点的理解**，为什么 Prefix-Tuning 更适合生成任务。

### 问题衍生
- Prefix-Tuning 为什么只注入 K 和 V，不注入 Q？
- Prefix-Tuning 相比 P-Tuning 增加了多少参数量？
- 如果要将 Prefix-Tuning 应用到编码 - 解码架构，应该加在哪些位置？

## 问题 3：P-Tuning 和 Prefix-Tuning 的核心区别

### 【问题】
P-Tuning 和 Prefix-Tuning 的核心区别

### 【回答】

| 对比维度 | P-Tuning（v1） | Prefix-Tuning |
|---------|---------------|---------------|
| **注入位置** | 仅输入层（Embedding） | 每一层（Layer-wise） |
| **注入方式** | 拼接到词嵌入前面 | 添加到 KV 向量序列中 |
| **影响深度** | 浅层，逐层传递会减弱 | 深层，直接影响每层注意力 |
| **控制力度** | 较弱，依赖信息向上传递 | 较强，直接修改 KV Cache |
| **适用任务** | NLU（理解类任务） | NLG（生成类任务） |
| **参数量** | 较少（仅一层） | 较多（每层都有） |

**本质区别**：P-Tuning 是"从外向内"的影响，通过在输入端添加提示，希望信息能逐层传递到深层；而 Prefix-Tuning 是"由内而外"的控制，直接在每一层的核心计算单元（注意力机制）中注入任务特定信息。

### 问题难点分析
这个问题的难点在于要从多个维度对比两种技术，并理解它们设计哲学上的根本差异。需要明白为什么不同的注入位置会导致适用任务的不同。

### 面试官考察点
- **对两种技术的系统性对比能力**，能否从多个维度分析。
- **对模型信息流的理解**，能否解释为什么注入位置影响适用任务。
- **技术选型能力**，能否根据任务类型选择合适的 Prompt Tuning 方法。

### 问题衍生
- 如果要结合两者的优点，你会怎么设计？
- 为什么 NLU 任务对深层控制的要求不如 NLG 任务高？
- P-Tuning v2 是如何改进 v1 的局限性的？
