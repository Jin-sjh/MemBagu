# SimHash 与 MinHash 去重算法

## 问题 0：什么是 SimHash 去重算法？

### 【问题】
什么是 SimHash 去重算法？

### 【回答】
SimHash 是一种用于近似重复检测的算法，它能将一个高维的文档或数据压缩成一个固定长度的指纹（fingerprint）或哈希值。这个指纹保留了原始数据的相似性信息，使得相似的文档具有相似的 SimHash 值（即它们之间的汉明距离很小）。

**核心原理：**
1. **特征提取**：将文档分词并计算每个词的权重（如 TF-IDF）
2. **哈希计算**：对每个特征词计算其哈希值
3. **加权合并**：根据权重对哈希值进行加权求和
4. **降维生成指纹**：将加权结果转换为二进制指纹

**汉明距离判断：**
- 两个文档的 SimHash 指纹之间的汉明距离越小，文档相似度越高
- 通常设定阈值（如汉明距离 ≤ 3）判断为重复或近似重复

**应用场景：**
- 网页去重（搜索引擎）
- 文档相似度检测
- 大规模数据去重

## 问题 1：什么是 MinHash 去重算法？

### 【问题】
什么是 MinHash 去重算法？

### 【回答】
MinHash 是一种高效估算两个集合之间 Jaccard 相似度的算法，特别适用于大规模数据集。它的核心思想是：如果两个集合的 Jaccard 相似度高，那么对这两个集合中的元素进行随机排列后，它们各自的"最小哈希值"很大概率是相同的。

**Jaccard 相似度：**
```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

**MinHash 核心思想：**
1. **随机哈希函数**：使用多个不同的随机哈希函数
2. **最小值计算**：对每个集合，计算所有元素经过哈希后的最小值
3. **相似度估计**：两个集合的最小哈希值相等的概率等于它们的 Jaccard 相似度

**算法优势：**
1. **高效性**：不需要计算完整的交集和并集，时间复杂度低
2. **可扩展性**：适合大规模数据集的去重和相似度计算
3. **局部敏感哈希（LSH）**：可与 LSH 结合，实现亚线性时间的相似项查找

**典型应用：**
- 大规模文本去重
- 推荐系统中的相似物品查找
- 网页去重与聚类
- DNA 序列相似性分析

## 问题 2：SimHash 与 MinHash 的区别是什么？

### 【问题】
SimHash 与 MinHash 的区别是什么？

### 【回答】
SimHash 和 MinHash 都是用于大规模数据去重和相似度检测的算法，但它们在原理、适用场景和特性上有明显区别：

| 特性 | SimHash | MinHash |
|------|---------|---------|
| **相似度度量** | 汉明距离（Hamming Distance） | Jaccard 相似度 |
| **数据表示** | 适用于高维向量（如文档的 TF-IDF 特征） | 适用于集合数据（如文档的词集合） |
| **局部敏感性** | 相似文档的哈希值相近 | 相似集合的最小哈希值相等概率高 |
| **计算复杂度** | 较低，适合实时计算 | 需要多个哈希函数，计算量相对较大 |
| **去重精度** | 适合检测近似重复（near-duplicate） | 适合检测集合相似性 |

**选择建议：**
- **使用 SimHash**：当需要检测内容几乎相同的文档（如网页去重），且要求计算速度快
- **使用 MinHash**：当需要比较集合相似度（如用户兴趣标签、文档关键词），且数据规模非常大时

**组合使用：**
在实际应用中，可以先用 MinHash 快速筛选出候选相似对，再用 SimHash 精确判断，实现高效且准确的去重系统。
