## 问题 1：Attention 和 Self-Attention 的区别

### 【问题】
Attention 和 Self-Attention 的区别

### 【回答】
注意力机制是一个比较宏大的概念，但是自注意力机制则是相对注意力机制提到的，一个相对具体的概念（自注意力机制是注意力机制的一个子集）。

- **注意力机制**：注意力机制约束你必须使用 Q、K 和 V（一般情况下 K == V），至于 Q、K、V 的来源，这个没有限制。

- **自注意力机制**：自注意力机制约束必须要使用 Q、K 和 V，同时 Q、K 与 V 同源，并且计算方式就是上述的点积。

- **交叉注意力机制**：交叉注意力机制约束必须使用 Q、K 和 V，并且 K 与 V 同源，Q 与 K 不同源。

### 问题难点分析
这个问题考察对注意力机制家族概念的理解和区分。难点在于要准确理解 Q、K、V 的来源和同源性概念，以及三种机制之间的包含关系。需要明确自注意力是注意力的特例，交叉注意力是另一种变体。

## 问题 2：在注意力分数计算中，为什么要除 √dk？

### 【问题】
在注意力分数计算中，为什么要除 √dk？

### 【回答】
**核心原因**：控制点积 QK^T 的方差，防止数值过大导致 Softmax 输出极端化（接近 0 或 1）。

**数学推导**：
- 点积 QK^T 的方差是 dk，标准差是 √dk。
- 为了将分布"拉回"标准尺度（标准差=1），需除以标准差（即 √dk），而非方差（dk）。
- 若除以 dk，方差会过小（1/dk），导致注意力分数过于集中，失去区分性。

**直观类比**：
假设原始点积值是"爆炸的火花"，除以 √dk 像是用"消防水雾"温和降温，而除以 dk 则是"冰桶挑战"，直接冻僵模型。

**效果**：Softmax 输入稳定，梯度传播更有效。

### 问题难点分析
这个问题考察对 Attention 机制数值稳定性的理解。难点在于要从方差角度理解缩放的必要性，而不仅仅是记住结论。需要理解当 Q 和 K 维度较大时，点积结果会很大，导致 Softmax 进入饱和区，梯度消失。

### 面试官考察点
- **对 Attention 机制数值稳定性的理解**。
- **概率论基础**，是否理解方差的计算和缩放的影响。
- **对 Softmax 函数特性的理解**，知道输入过大会导致梯度消失。

## 问题 2：为什么长文本推理时 Attention 计算会成为瓶颈？如何解决？

### 【问题】
为什么长文本推理时 Attention 计算会成为瓶颈？如何解决？

### 【回答】
Attention 机制的计算复杂度是 O(n²)（n 为序列长度），随着文本变长，计算量和显存占用呈平方级增长，因此成为瓶颈。具体来说，自注意力需要计算每个 token 对所有 token 的注意力分数，导致显存和计算时间爆炸。

解决方案包括：

1. **稀疏注意力**：限制每个 token 只关注局部窗口或少数全局 token，如 Longformer、BigBird。

2. **线性注意力**：将 softmax 注意力近似为核函数形式，使复杂度降为 O(n)，如 Performer。

3. **分段处理**：将长文本切分成多个片段，分别处理后再融合，如 Transformer-XL。

4. **改进推理策略**：使用 KV 缓存，避免重复计算；或采用流式推理，逐步生成。

5. **硬件优化**：使用 FlashAttention 等 IO 感知的注意力实现，减少显存读写。

### 问题难点分析
这个问题既考察理论基础（复杂度分析），又考察工程实践（解决方案）。难点在于不仅要列出方法，还要理解每种方法的 trade-off，比如稀疏注意力可能损失全局信息，线性注意力可能带来精度下降。

- **对 Transformer 核心瓶颈的深刻理解**。
- **是否关注长文本建模的前沿研究**，如 Longformer、FlashAttention。
- **工程权衡能力**，在资源限制下如何选择最合适的方案。

## 问题：在 Transformer 中，X 是哪里来的？

### 【问题】
在 Transformer 中，X 是哪里来的？

### 【回答】
Transformer 的输入 X 是以下三部分的加和：

**1. Token Embeddings（词嵌入）**
- **来源**：直接通过嵌入层（Embedding Layer）从 token 映射到向量，不是通过独热编码 + Word2Vec。
  - 例如，BERT 使用 WordPiece 分词，每个 token 对应一个可学习的嵌入向量。
  - 原始 Transformer 使用可训练的嵌入层（类似 Word2Vec，但端到端学习）。
- **维度**：d_model（如 512 维）。

**2. Positional Encodings（位置编码）**
- **作用**：注入序列的位置信息（因为自注意力本身是无序的）。
- **方法**：
  - 原始 Transformer 使用固定的正弦/余弦函数生成位置编码。
  - BERT 等模型改用可学习的位置嵌入（Position Embeddings）。
- **操作**：与词嵌入直接相加。

**3. Segment Embeddings（可选，如 BERT）**
- 在需要区分句子时（如 NSP 任务），加入句子标记的嵌入（如 [CLS]、[SEP]）。

也就是说，X 是词嵌入和位置编码的总和，那这里的词嵌入是怎么做的呢？

**(1) 输入分词（Tokenization）**
- 原始 Transformer 的输入是子词（Subword）或单词级别的 token，例如通过 Byte-Pair Encoding (BPE) 分词的序列（如将"playing"拆分为["play", "ing"]）。
- 每个 token 会被映射为一个唯一的整数 ID（词汇表索引）。

**(2) 词嵌入（Token Embeddings）**
- **嵌入矩阵**：模型维护一个可训练的权重矩阵 E ∈ R^(V×d_model)，其中：
  - V 是词汇表大小（如 37000 个 BPE 子词）。
  - d_model 是词嵌入维度（如 512）。
- **查找表（Lookup）**：
  根据 token 的 ID，从矩阵 E 中取出对应的行向量作为该 token 的嵌入表示。
  
  公式：E_token = E[token_id] ∈ R^d_model

- **本质**：词嵌入矩阵 E 是一个可学习的参数，通过训练数据优化，而非静态的预训练嵌入（如 Word2Vec）。

### 问题难点分析
这个问题考察对 Transformer 输入表示的完整理解。难点在于要清楚 Transformer 输入的三部分组成，以及词嵌入的具体实现方式。需要理解词嵌入是通过查找可学习矩阵实现的，而不是使用预训练的 Word2Vec。

### 面试官考察点
- **对 Transformer 输入表示的完整理解**，是否清楚三部分的来源和作用。
- **对词嵌入实现细节的掌握**，能否说明白从 token 到向量的完整流程。
- **对可学习参数与预训练嵌入的区别**，理解端到端学习的优势。

## 问题：Self Attention 的优势是什么？

### 【问题】
Self Attention 的优势是什么？

### 【回答】
**1. 捕获长距离依赖**
通过上述两幅图，很明显的可以看出，引入 Self Attention 后会更容易捕获句子中长距离的相互依赖的特征，因为 Self Attention 在计算过程中会直接将句子中任意两个单词的联系通过一个计算步骤直接联系起来，所以远距离依赖特征之间的距离被极大缩短，有利于有效地利用这些特征。

**2. 并行计算能力**
除此之外，Self-Attention 对于一句话中的每个单词都可以单独的进行 Attention 值的计算，也就是说 Self Attention 对计算的并行性也有直接帮助作用，而对于必须得依次序列计算的 RNN 而言，是无法做到并行计算的。

### 问题难点分析
这个问题考察对 Self Attention 机制核心优势的理解。难点在于要从两个维度理解 Self Attention 相比 RNN 的改进：一是长距离依赖的捕获能力，二是并行计算效率。需要理解 Self Attention 如何通过一步计算建立任意两个 token 之间的直接联系，以及这种设计如何解决 RNN 的序列依赖问题。

## 问题：在使用 Transformer 生成文本的时候，为什么需要引入 masked self Attention？

### 【问题】
在使用 Transformer 生成文本的时候，为什么需要引入 masked self Attention？

### 【回答】
Masked Self-Attention 的核心作用是保持生成过程的因果性，确保模型在训练和推理时行为一致，从而生成连贯、合理的序列。这是 Transformer 能够成功用于文本生成任务（如 GPT、机器翻译）的关键设计之一。

Transformer 的文本生成本质是逐步、自回归的预测过程，依赖 Masked Self-Attention 实现因果性。模型在每一步：

1. **仅关注已生成的词（通过掩码）**。

2. **计算下一个词的概率分布**。

3. **选择词并循环迭代**，直到生成完整序列。

### 问题难点分析
这个问题考察对 Transformer 解码器中 Masked Self-Attention 机制的理解。难点在于要理解因果性约束的重要性，以及为什么在文本生成任务中需要防止模型看到未来的信息。需要明确 Masked Self-Attention 通过遮蔽未来位置，确保每个位置只能关注之前的位置，从而保证自回归生成的正确性。

### 面试官考察点
- **对 Transformer 解码器架构的理解**，清楚 Encoder-Decoder 与 Decoder-only 的区别。
- **对因果性约束的理解**，明白为什么文本生成需要防止信息泄露。
- **对训练与推理一致性的理解**，能否说明白 Masked 机制如何保证两种场景行为相同。

## 问题：为什么 Decoder 要输入偏移？

### 【问题】
为什么 Decoder 要输入偏移？核心是对齐自回归生成。

### 【回答】
推理时，Decoder 必须逐步生成 token（如首先生成 `<sos>`，然后生成"I"，再生成"love"......），每一步只能看到已生成的部分。训练时，如果直接输入完整目标序列（如 `<sos> I love AI <eos>`），模型会"看到"整个句子，导致它直接复制未来 token，而无法学会"根据历史预测未来"。因此需要将输入序列右移一位，确保模型在预测第 t 个 token 时，只能看到前 t-1 个 token（丢掉最后一个 `<eos>`）。

### 问题难点分析
这个问题考察对 Transformer Decoder 自回归训练机制的理解。难点在于要理解训练时输入偏移的核心目的是模拟推理时的逐步生成过程，防止模型作弊看到未来信息。需要明确偏移操作如何保证因果性，以及为什么这样做能让模型学会根据历史预测未来。

## 问题：Transformer 中 Encoder 和 Decoder 的 Self-Attention 有什么区别？

### 【问题】
Transformer 中 Encoder 和 Decoder 的 Self-Attention 有什么区别？

### 【回答】

| 模块 | Self-Attention 的输入 | 是否掩码 | 目的 |
|------|---------------------|---------|------|
| Encoder | 输入序列的所有 token | 无 | 编码源序列内部关系 |
| Decoder | 已生成的目标序列（过去 token + 自己） | 有 | 避免信息泄漏，自回归生成 |

**Encoder 的 Self-Attention：**
Encoder 当前层的输入序列（如果是第一层，则是词嵌入 + 位置编码；其他层是前一层的输出）。每个 token 会关注输入序列中的所有 token（包括自己），无掩码。

**Decoder 的 Self-Attention：**
Decoder 当前层的已生成序列（训练时是目标序列的掩码输入，推理时是逐步生成的序列）。每个 token 只能关注它之前的位置（包括自己），通过掩码屏蔽未来信息。

### 问题难点分析
这个问题考察对 Transformer 架构中 Encoder 和 Decoder 的 Self-Attention 机制差异的理解。难点在于要清楚两者的输入来源不同、掩码策略不同、以及设计目的不同。需要理解 Encoder 需要完整的双向上下文信息来编码源序列，而 Decoder 需要保持因果性以进行自回归生成。

### 面试官考察点
- **对 Transformer 整体架构的理解**，是否清楚 Encoder 和 Decoder 的不同职责。
- **对 Self-Attention 掩码机制的理解**，能否说明白为什么 Decoder 需要掩码而 Encoder 不需要。
- **对训练与推理过程的理解**，清楚 Decoder 在两种场景下的输入差异。

## 问题：多头注意力中"分头"是对输入 X 切分吗？

### 【问题】
多头注意力中"分头"是对输入 X 切分吗？

### 【回答】
不是。

- **分头的对象**：对权重矩阵 W^Q、W^K、W^V 进行分块（如拆分为 8 个 [512,64] 的子矩阵），而非直接切分输入 X。

- **计算逻辑**：输入 X 保持完整形状（如 [32,100,512]），通过单次矩阵乘法隐式分头，再通过 reshape/transpose 操作拆分。

### 问题难点分析
这个问题考察对 Multi-Head Attention 实现机制的理解。难点在于要清楚分头操作实际上是对权重矩阵的分块，而不是对输入 X 的直接切分。需要理解通过权重矩阵的分块设计，配合 reshape 和 transpose 操作，可以高效地实现多头计算。

## 问题：Layer Norm 与 Batch Norm 的核心区别是什么？

### 【问题】
Layer Norm 与 Batch Norm 的核心区别是什么？

### 【回答】
**1. 归一化的维度不同**

| 方法 | 归一化维度 | 适用场景 |
|------|-----------|---------|
| Batch Norm | 对同一特征的所有样本（Batch 维度）归一化 | 小批次数据较稳定（如 CNN） |
| Layer Norm | 对同一样本的所有特征（Channel 维度）归一化 | 变长序列、小批次或单样本（如 Transformer） |

**2. Batch Norm (BN)**
- **计算方式**：计算一个 Batch 内所有样本的同一特征的均值和方差。
- **输入维度**：`[batch_size, channels, height, width]`，对每个 channel 的 `[batch_size, height, width]` 数据归一化。
- **问题**：对 Batch Size 敏感，若 Batch Size 小（如 1），均值和方差估计不准；且不适用于变长序列（如 NLP 任务）。

**3. Layer Norm (LN)**
- **计算方式**：计算单个样本的所有特征的均值和方差。
- **输入维度**：`[batch_size, seq_len, hidden_dim]`，对每个样本的 `[seq_len, hidden_dim]` 数据归一化。
- **优势**：不受 Batch Size 影响，对变长序列友好。

## 问题：为什么 Transformer 使用 Layer Norm，而不是 Batch Norm？

### 【问题】
为什么 Transformer 使用 Layer Norm，而不是 Batch Norm？

### 【回答】
**1. 处理变长序列**
- Transformer 的输入是变长文本（如句子），每个样本的 `seq_len` 可能不同。
- BN 需要固定 `seq_len` 或填充后计算，会引入噪声。
- LN 直接对每个样本独立归一化，天热支持变长输入。

**2. 小批次或单样本训练的稳定性**
- Transformer 常需处理小批次（甚至单样本）训练（如大模型微调）。
- BN 在小 Batch 下统计量估计不准，导致性能下降。
- LN 无此限制，即使 Batch Size=1 也能稳定工作。

**3. 与自注意力机制的兼容性**
- 自注意力机制本身对输入尺度的敏感性较低（因 Softmax 的归一化性质），但残差连接需要稳定的数值范围。
- LN 在残差连接后使用（如 Post-LN），可防止梯度爆炸/消失。
- BN 会对序列内不同位置的统计量差异破坏注意力权重（尤其是长序列）。

**4. 推理一致性**
- Transformer 常常处理动态生成的序列（如机器翻译逐词生成）。
- BN 的滑动平均在推理时可能不适应新序列长度。
- LN 的逐样本计算无需调整，保证一致性。

### 问题难点分析
这个问题考察对归一化技术在不同架构中适用性的理解。难点在于要从多个维度理解 Layer Norm 相比 Batch Norm 更适合 Transformer 的原因：包括变长序列处理、小 Batch 训练、与自注意力机制的兼容性、以及推理一致性。需要清楚 BN 和 LN 在归一化维度上的本质区别，以及这种区别带来的实际影响。

## 问题：Transformer 的输出结果 - 各模块计算量分析

### 【问题】
Transformer 的输出结果 - 各模块计算量分析

### 【回答】

| 模块 | 单步计算量 | 全局计算量 | 原因 |
|------|-----------|-----------|------|
| Encoder Self-Attention | O(n²) | O(n²) | 所有 token 同时交互 |
| Decoder Self-Attention | O(n) | O(n²) | 仅新 token 与历史交互（逐步累计） |
| Cross-Attention | O(m) | O(n·m) | 新 token 的 Q 与所有 Encoder K/V 交互 |

**详细说明：**

**1. Encoder Self-Attention**
- **单步计算量**：O(n²)
- **全局计算量**：O(n²)
- **原因**：Encoder 的 Self-Attention 是并行处理整个序列，所有 token 同时交互，不存在"单步生成"的概念（Decoder 才有）。因此单步和全局计算量相同，都是 O(n²)。

**2. Decoder Self-Attention**
- **单步计算量**：O(n)
- **全局计算量**：O(n²)
- **原因**：Decoder 是自回归生成，每次只生成一个新 token。单步计算时，新 token 只需与之前的 n 个历史 token 交互，所以是 O(n)。但生成整个序列（n 个 token）时，逐步累计的计算量就是 O(n²)。

**3. Cross-Attention**
- **单步计算量**：O(m)
- **全局计算量**：O(n·m)
- **原因**：Cross-Attention 中，Decoder 的新 token 的 Q 需要与 Encoder 输出的所有 m 个 token 的 K/V 进行交互。单步是 O(m)，生成 n 个 token 的全局计算量就是 O(n·m)。

### 问题难点分析
这个问题考察对 Transformer 各模块计算复杂度的深入理解。难点在于要区分 Encoder 和 Decoder 在计算模式上的本质差异：Encoder 是并行处理，Decoder 是序列生成。需要理解"单步计算量"和"全局计算量"的概念差异，以及不同 Attention 机制的计算特点。

  这种操作无法生成输入中不存在的新特征模式（例如无法实现 $x_j^2$、$\sin(x_j)$ 等非线性变换）。

## 问题：在 Transformer 中，Decoder 在机器翻译任务中，Q 是要生成的 token 的 Q 吗，还是上一个 token 的 Q？KV 是全部源句的 KV 吗？

在 Transformer 的 Decoder 中，Decoder 在机器翻译任务中，Q 是要生成的 token 的 Q 吗，还是上一个 token 的 Q？KV 是全部源句的 KV 吗？

### 【回答】
在 Transformer 的 Decoder 中，Query (Q)、Key (K)、Value (V) 的来源和角色需要根据注意力机制的类型（自注意力 vs. 交叉注意力）区分：

**Q 的来源：**
- **掩码自注意力中的 Q**：当前步所有已生成目标 token 的隐藏状态（如生成"love"时，Q 来自 <sos> 和"I"）。
- **交叉注意力中的 Q**：当前步的目标 token 隐藏状态（如生成"love"时，Q 是"I"的隐藏状态）。

**K/V 的来源：**
- **掩码自注意力中的 K/V**：来自已生成目标 token（与 Q 相同，但 K/V 会被缓存复用）。
- **交叉注意力中的 K/V**：始终来自 Encoder 输出的源句编码（全句 KV，固定不变）。

**K/V 是否包含全部源句？**
- **是的**：交叉注意力中的 K/V 是 Encoder 对整个源句（如"我 爱 AI"）的编码，Decoder 在每一步生成时都会访问完整的源句 KV。

### 问题难点分析
这个问题考察对 Decoder 中两种注意力机制（掩码自注意力和交叉注意力）的 Q、K、V 来源的准确理解。难点在于要区分两种注意力机制中 Q、K、V 的不同来源：掩码自注意力中 Q、K、V 都来自已生成的目标序列，而交叉注意力中 Q 来自目标序列、K/V 来自源序列。需要理解 Decoder 生成的自回归特性，以及交叉注意力如何实现源句到目标句的信息传递。

## 问题：Transformer 的位置编码有什么劣势？

### 【问题】
Transformer 的位置编码有什么劣势？

### 【回答】
**1. 三角函数位置编码（Sinusoidal PE）的点积结果仅依赖两个位置的绝对距离（Δt），而无法区分方向（前向或后向）。**

原始 Transformer 使用的正弦/余弦位置编码，其点积结果只与两个 token 之间的相对距离有关，但无法区分前后方向。例如，位置 t 和位置 t+2 的关系，与位置 t 和位置 t-2 的关系在点积中是相同的，这限制了模型对方向性信息的捕捉。

**2. 相对位置信息在 self-attention 处线性变化后消失。**

位置编码是与词嵌入相加后输入到 Self-Attention 的，在经过 Q、K、V 的线性投影以及后续的注意力计算后，原始的相对位置信息会被线性变换"稀释"或"掩盖"，导致位置信息在深层网络中逐渐弱化。

### 问题难点分析
这个问题考察对 Transformer 位置编码机制局限性的深入理解。难点在于要从两个维度理解位置编码的不足：一是 Sinusoidal PE 在方向性上的缺陷，二是位置信息在线性变换中的衰减问题。需要理解这些劣势如何影响了模型对位置敏感任务的表现，以及为什么后续研究提出了 RoPE、ALiBi 等改进方案。

## 问题：什么是 RoPE（旋转位置编码）？它的核心思想是什么？

### 【问题】
什么是 RoPE（旋转位置编码）？它的核心思想是什么？

### 【回答】
RoPE（Rotary Position Embedding，旋转位置编码）实现了绝对位置编码和相对位置编码的统一，它通过绝对位置编码的形式，实现了相对位置编码的效果。

RoPE 将输入序列的位置信息通过旋转操作嵌入到 self-attention 的计算中，不同位置的 token 可以有不同的旋转角度，从而嵌入位置信息，从而增强模型对长序列和相对位置的处理能力。

**核心机制**：
1. **旋转矩阵**：RoPE 通过将 token 的 Q 和 K 向量与位置相关的旋转矩阵相乘，将位置信息注入到注意力计算中。

2. **相对位置感知**：两个 token 的注意力分数只依赖于它们的相对位置（m - n），而不是绝对位置，这使得模型能够更好地捕捉相对位置关系。

3. **外推性强**：相比传统的正弦位置编码，RoPE 具有更好的长度外推能力，能够处理超出训练长度的序列。

**数学形式**：
对于位置 m 的 token，其查询向量 Q_m 经过旋转编码后变为：
Q_m = R(m) · Q

其中 R(m) 是位置 m 对应的旋转矩阵。在计算注意力分数时：
Q_m · K_n = Q^T · R(m - n) · K

可以看到，注意力分数只依赖于相对位置 (m - n)。

### 问题难点分析
这个问题考察对 RoPE 位置编码机制的全面理解。难点在于要理解 RoPE 如何巧妙地通过旋转操作将绝对位置编码转化为相对位置编码的效果，以及这种设计带来的外推性优势。需要理解 RoPE 的数学原理和工程价值。

- **对旋转操作数学原理的理解**，清楚 RoPE 如何在注意力计算中实现相对位置感知。
- **对位置编码演进脉络的把握**，理解 RoPE 相比传统方法的改进之处。
- **对工程实践的了解**，是否知道 RoPE 在现代大模型（如 LLaMA、PaLM）中的广泛应用。

## 问题：为什么 RoPE 能外推到更长文本？

### 【问题】
为什么 RoPE 能外推到更长文本？

### 【回答】
**传统位置编码的问题：**

- **绝对位置编码（如 BERT）**：位置编号 1,2,...,512，遇到 513 就超出范围。

- **相对位置编码**：需要预设最大距离（如最多支持 512 个词的相对距离）。

**RoPE 的巧妙之处：**

- **位置信息转化为旋转角度**：模型只关心词与词之间的相对旋转角度差（如 m 词和 n 词之间的角度差是 (m-n)θ），而不是绝对位置。

- **无限旋转**：即使文本长度从 1000 增长到 2000，模型只需要知道新的位置差（如 2000-1000=1000）对应的旋转角度是 1000θ，而这个计算方式和训练时完全一致。

这个问题考察对 RoPE 位置编码外推性优势的深入理解。难点在于要理解 RoPE 如何将绝对位置信息转化为相对旋转角度差，从而实现长度外推。需要明白传统位置编码受限于预定义的最大长度，而 RoPE 通过旋转角度的线性增长特性，可以自然地处理任意长度的序列。

## 问题：什么是 KV Cache？它的核心原理是什么？

### 【问题】
什么是 KV Cache？它的核心原理是什么？

### 【回答】
**KV Cache（键值缓存）** 是 Transformer 模型在自回归生成推理时的核心优化技术，用于避免重复计算历史 token 的 Key 和 Value 矩阵。

**核心原理：**

KV Cache 缓存的就是 K 矩阵和 V 矩阵。

**计算注意力分数的流程（自回归生成时，第 t 步的注意力计算）：**

**1. 生成当前 token 的 Q/K/V：**
- 当前 token 的输入经过线性层，得到 Q_t、K_t、V_t。

**2. 读取历史 KV Cache：**
- 加载之前所有历史 token 的 K_{1:t-1} 和 V_{1:t-1}。

**3. 计算注意力分数：**
- QK_t = Q_t @ concat(K_{1:t-1}, K_t)^T（动态计算，无需缓存）。

**4. 加权求和：**
- Output_t = softmax(QK_t) @ concat(V_{1:t-1}, V_t)（动态计算，无需缓存）。

**核心价值：**
- **避免重复计算**：历史 token 的 K/V 只需计算一次，后续复用缓存值。
- **降低计算复杂度**：将每步从 O(t²) 降低到 O(t)，大幅提升推理效率。
- **减少显存访问**：避免重复从显存读取历史 token 进行计算。

### 问题难点分析
这个问题考察对 Transformer 推理优化核心技术的理解。难点在于要理解 KV Cache 如何通过缓存历史 K/V 矩阵来避免重复计算，以及这种优化如何将计算复杂度从 O(t²) 降低到 O(t)。需要清楚自回归生成时每一步只需要计算当前 token 的 Q/K/V，而历史 K/V 可以从缓存中直接读取。
