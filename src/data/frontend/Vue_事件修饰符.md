# Vue 事件修饰符

## 【问题】
事件修饰符（如.native）底层做了哪些处理？

## 【回答】

### 1. 通用事件修饰符（.stop/.prevent/.once 等）

底层本质是**高阶函数包装回调**：在执行用户定义的回调函数前，先执行修饰符对应的逻辑（如阻止冒泡、阻止默认行为、只执行一次），再调用原回调，是对原生事件行为的封装增强。

### 2. .native 修饰符

专门用于给**Vue 组件绑定原生 DOM 事件**，底层核心处理：

- 强制**跳过 Vue 自定义事件的发布订阅机制**；

- 直接获取组件实例对应的**根 DOM 元素**；

- 调用原生 `addEventListener` 给该根元素绑定原生事件，实现直接监听组件 DOM 原生事件的效果。
