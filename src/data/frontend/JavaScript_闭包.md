# 闭包

## 【问题】
什么是闭包？

## 【回答】
闭包是指：函数能够访问并记住自己定义时的词法作用域，即使这个函数在定义作用域之外执行，依然能访问外层变量。

简单说：内部函数 + 它引用的外层作用域，一起形成了闭包。

## 【问题】
什么是闭包？闭包有哪些应用场景？

## 【回答】
**闭包：** 函数与其词法环境的组合，即使函数在其词法环境之外执行，也能访问该环境中的变量。

**特点：**
1. 函数嵌套函数
2. 内部函数可以访问外部函数的变量
3. 外部函数的变量会被保存在内存中

**应用场景：**
1. 私有变量
2. 函数工厂
3. 模块模式
4. 记忆化
5. 事件处理

## 【问题】
闭包和作用域链是什么关系？

## 【回答】
- **作用域链是闭包的基础**：没有作用域链，内部函数就无法访问外层变量。
- **闭包是作用域链的延伸**：正常函数执行完作用域会销毁，而闭包让内部函数保留外层作用域不销毁，带到外部也能正常访问。

## 【问题】
闭包有什么用？

## 【回答】
1. **私有化变量**，避免全局污染
2. **保存变量状态**，实现计数器、缓存等
3. 在定时器、循环、回调中**保留当前作用域的变量**

## 【问题】
闭包有什么缺点？

## 【回答】
闭包会让外层作用域的变量**无法被垃圾回收**，使用不当可能造成**内存占用过高**、**内存泄漏**，用完需要手动置空释放。

## 【问题】
什么是闭包？

## 【回答】
闭包是指**内部函数引用了外部函数作用域中的变量或参数**，使得外部函数执行完毕后，其作用域不会被垃圾回收，内部函数依然能访问该作用域变量的现象。

**总结**：内部函数引用外部变量 → 作用域不销毁 → 形成闭包。

## 【问题】
闭包的原理是什么？

## 【回答】
JS 存在函数作用域，函数执行会创建执行上下文。正常情况下函数执行完上下文会被销毁，但如果内部函数引用了外部函数变量，该外部作用域会被保留，内部函数在任何位置执行都能访问这些变量。

**总结**：引用保留作用域，延长变量生命周期。

## 【问题】
闭包有哪些应用场景？

## 【回答】
- 实现私有变量，避免全局污染
- 实现持久化状态，如计数器
- 解决循环中 var 变量共享问题
- 实现模块化、防抖、节流、函数柯里化

**总结**：封装私有变量、保存状态、解决作用域共享问题。

## 【问题】
闭包的优缺点是什么？

## 【回答】
**优点**：不产生全局变量，实现属性私有化。

**缺点**：闭包中的数据会常驻内存，在不用的时候需要删除，否则会导致内存溢出（内存泄漏）。

---

## 【问题】
闭包的优缺点？

## 【回答】
**优点**：变量私有化、数据持久化、代码模块化、避免全局污染。

**缺点**：变量常驻内存，使用不当易造成内存泄漏，过多使用影响性能。

**总结**：优在封装与状态，弊在内存占用与泄漏风险。

## 【问题】
闭包造成的内存泄漏如何解决？

## 【回答】
使用完闭包后，手动解除引用，将引用闭包的变量赋值为 `null`，让垃圾回收机制回收对应的作用域。

**总结**：不用置 null，释放内存。

## 【问题】
JS 闭包是什么？一般在什么场景下会用？

## 【回答】
**定义：**
闭包是函数与其周围词法环境的组合，当一个内部函数可以访问外部函数的作用域，且外部函数执行完毕后，内部函数仍然保持对外部变量的引用，这就形成了闭包。

简单说：一个函数"记住"了它定义时的作用域，即使它在定义作用域之外调用，也能访问该作用域的变量。

**核心特点：**
1. 可以访问外部函数的变量
2. 外部函数执行完毕后，变量不会被垃圾回收，会被闭包继续持有
3. 实现变量私有化，避免全局污染

**常见使用场景：**

1. **封装私有变量**
   比如实现计数器，让变量只能通过特定暴露的方法修改，无法直接访问：
   ```javascript
   function createCounter() {
     let count = 0;
     return {
       increment() {
         count++;
         return count;
       },
       getCount() {
         return count;
       }
     };
   }
   const counter = createCounter();
   counter.increment();
   console.log(counter.getCount());
   ```

2. **模块化开发（早期）**
   在 ES6 模块出现之前，使用闭包实现模块化，暴露公共 API，隐藏内部实现：
   ```javascript
   var module = (function() {
     var privateVar = 'secret';
     function privateMethod() {
       return privateVar;
     }
     return {
       publicMethod: function() {
         return privateMethod();
       }
     };
   })();
   ```

3. **函数柯里化 / 偏函数**
   固定部分参数，生成新函数：
   ```javascript
   function add(a) {
     return function(b) {
       return a + b;
     };
   }
   const add5 = add(5);
   console.log(add5(3)); // 8
   ```

4. **异步回调中保留上下文**
   比如在循环中绑定事件时，用闭包保留循环变量的值，避免变量被覆盖：
   ```javascript
   for (var i = 0; i < 5; i++) {
     (function(j) {
       setTimeout(() => console.log(j), 1000);
     })(i);
   }
   ```

5. **防抖 / 节流函数**
   用闭包保存定时器 ID，上次执行时间等状态变量。

## 【问题】
为什么需要闭包，而不是全局变量？

## 【回答】
闭包的核心价值是**实现变量的私有性、封装性和持久化**，而全局变量存在严重的缺陷，两者不可替代：

**1. 全局变量的致命问题**
- 污染全局命名空间，容易被意外修改、覆盖，引发不可预期的 bug；
- 生命周期贯穿整个应用，无法按需销毁，造成内存泄漏风险；
- 无法实现模块化、封装性，代码可维护性、可复用性极差。

**2. 闭包的核心优势**
- **私有变量保护**：闭包内的变量仅能通过内部暴露的方法访问，外部无法直接修改，避免了全局污染；
- **状态持久化**：闭包可以让函数执行完后，其作用域内的变量不被垃圾回收，实现状态的缓存 / 记忆；
- **模块化开发**：是 JS 实现模块化（IIFE、CommonJS、ES Module）的底层原理，可实现高内聚、低耦合的代码结构。

**面试加分点**
- 闭包的本质：**函数嵌套函数，内部函数引用外部函数的变量，导致外部函数的作用域无法被销毁**；
- 适用场景：防抖 / 节流、柯里化、模块化、单例模式、私有属性实现；
- 对比：全局变量是「共享的、无保护的」，闭包是「私有的、受控的」。
