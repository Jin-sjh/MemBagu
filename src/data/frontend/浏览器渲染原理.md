【问题】请用一句话概括浏览器渲染的完整流程

【回答】浏览器从接收 HTML、CSS、JS 资源开始，依次完成解析构建、布局计算、像素绘制、图层合成，最终将页面内容渲染到屏幕上。

【问题】详细说说浏览器渲染的具体步骤

【回答】浏览器渲染主要分为 6 个核心步骤，具体如下：

1. 解析 HTML，构建 DOM 树：浏览器将 HTML 文本解析成浏览器可识别的 DOM 节点，按层级关系生成完整的 DOM 树，这是页面的结构基础。

2. 解析 CSS，构建 CSSOM 树：解析页面中所有 CSS 样式（包括内联样式、外部样式表、内嵌 style 标签），根据每个节点的样式规则，生成 CSSOM 树，明确每个元素的样式信息。

3. 生成渲染树（Render Tree）：将 DOM 树和 CSSOM 树结合，只筛选需要显示的节点（排除 display: none 等隐藏节点），构建出渲染树，包含节点的属性和样式信息。

4. 布局（Layout/Reflow）：基于渲染树，计算每个可见节点的几何位置、大小、边距等布局信息，确定元素在页面中的具体位置和尺寸。

5. 绘制（Paint/Repaint）：根据布局结果，将渲染树中的节点转化为屏幕上的像素，填充颜色、背景、边框、阴影等视觉属性，这个过程会生成多个图层。

6. 合成（Composite Layers）：浏览器将不同图层按正确的层级顺序合并，处理图层的重叠、遮挡关系，最终将合成后的画面显示在屏幕上。

【问题】JavaScript 和 CSS 会对渲染流程产生什么影响？如何优化？

【回答】

**影响**

1. CSS 的影响：CSS 会阻塞渲染资源，只有 CSSOM 树构建完成，浏览器才会开始渲染流程。如果 CSS 加载缓慢，会导致页面白屏或渲染延迟。

2. JS 的影响：JS 会阻塞 HTML 解析和渲染的双重阻塞资源，当浏览器遇到 `<script>` 标签（无 defer/async 属性时），会暂停 HTML 解析和渲染，先执行 JS 代码。因为 JS 可能会修改 DOM 和 CSSOM，所以浏览器必须等待 JS 执行完毕才能继续渲染。

**优化方案**

1. CSS 优化：
   - 压缩 CSS 代码，减少文件体积，加快加载速度。
   - 拆分 CSS，按页面模块按需加载，避免加载无关样式。
   - 避免使用复杂 CSS 选择器，减少 CSSOM 构建耗时。

2. JS 优化：
   - 延迟加载非关键 JS：给 `<script>` 标签添加 defer（按文档顺序执行，下载不阻塞解析）或 async（下载完立即执行，不保证执行顺序）属性。
   - 将 `<script>` 标签放在 `</body>` 底部，等 HTML 解析完成后再加载执行 JS。
   - 精简 JS 代码，减少不必要的 DOM 操作，避免重复触发重排重绘。

【问题】什么是重排（Reflow）和重绘（Repaint）？如何减少它们的性能消耗？

【回答】

**重排（Reflow）**

- 定义：当修改元素的几何属性（如宽高、位置、边距、字体大小、页面尺寸变化等）时，浏览器需要重新计算元素的布局信息，更新渲染树，这个过程就是重排。

- 触发场景：添加/删除可见 DOM 元素、修改元素尺寸、调整窗口大小、获取布局属性（如 offsetTop、clientWidth）等。

- 性能影响：重排会触发整个渲染流程（布局→绘制→合成），消耗性能最大。

**重绘（Repaint）**

- 定义：当修改元素的视觉属性（如颜色、背景、边框样式、阴影等），不影响布局，仅需重新绘制像素的过程，就是重绘。

- 触发场景：修改 color、background-color、border-color、opacity 等属性。

- 性能影响：重绘只触发绘制 + 合成，性能消耗低于重排。

**优化策略**

1. 批量修改 DOM：先将元素隐藏（display: none），再批量修改样式，最后重新显示，减少重排次数。

2. 避免频繁读取布局属性：读取 offsetTop、scrollTop 等属性会触发浏览器强制同步布局，应该读取后再修改。

3. 使用 CSS3 硬件加速：将 transform、opacity 等属性提升为独立图层，仅触发合成操作，不触发重排和重绘，性能最优。

4. 减少复杂的 CSS 选择器：避免深层嵌套、通配符选择器，减少浏览器渲染树匹配耗时。

【问题】浏览器的渲染进程有哪些线程？

【回答】

1. GUI 渲染线程
   解析 HTML/CSS，构建 DOM 树、CSSOM 树、渲染树，负责页面布局与绘制，与 JS 引擎线程互斥。

2. JavaScript 引擎线程
   解析执行 JS 代码，是单线程，长时间运行会阻塞页面渲染。

3. 事件触发线程
   管理事件循环，将鼠标、键盘、异步回调等事件加入任务队列。

4. 定时器触发线程
   处理 setTimeout、setInterval 计时，计时完毕将回调推入任务队列。

5. 异步 HTTP 请求线程
   处理 XHR/fetch 网络请求，请求完成后将回调放入任务队列。
