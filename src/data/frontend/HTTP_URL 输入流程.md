# 输入 URL 会发生什么

## 8. 输入 URL 会发生什么

### 【问题】
浏览器输入 URL 后发生了什么？

### 【回答】

浏览器输入 URL 后，会按以下流程完成从输入到页面渲染的全过程：

**1. 输入解析与缓存检查**
- 浏览器解析输入内容，判断是 URL 还是搜索词；
- 检查浏览器缓存（Memory Cache → Disk Cache），如果命中且未过期，直接读取缓存资源。

**2. DNS 解析**
- 若未命中缓存，向 DNS 服务器查询域名对应的 IP 地址；
- 流程：浏览器缓存 → 操作系统缓存 → 路由器缓存 → ISP DNS 服务器 → 递归/迭代查询。

**3. 建立 TCP 连接**
- 根据 IP 地址，通过三次握手与服务器建立 TCP 连接：
  - 客户端发送 SYN 包（请求连接）；
  - 服务器回复 SYN+ACK 包（同意连接）；
  - 客户端回送 ACK 包（连接建立完成）。
- 若使用 HTTPS，在 TCP 连接后进行 TLS 握手，协商加密套件、验证证书，建立安全连接。

**4. 发送 HTTP 请求**
- 浏览器构造 HTTP 请求报文（请求行、请求头、请求体），发送给服务器；
- 请求头包含 Cookie、User-Agent、Accept-Encoding 等信息。

**5. 服务器处理请求并返回响应**
- 服务器接收请求，解析请求，匹配后端服务（Nginx 反向代理到应用服务器）；
- 应用服务器处理业务逻辑，生成响应报文（状态码、响应头、响应体），返回给浏览器。

**6. 浏览器解析响应与渲染页面**
- 解析 HTML，构建 DOM 树；
- 解析 CSS，构建 CSSOM 树；
- 结合 DOM 与 CSSOM 生成渲染树（Render Tree）；
- 布局（Layout/Reflow）：计算每个节点的位置和大小；
- 绘制（Paint）：将节点绘制到屏幕；
- 合成（Composite）：将页面分层，交给 GPU 合成显示。

**7. 关闭连接**
- 请求完成后，通过四次挥手关闭 TCP 连接（或保持长连接复用）。
