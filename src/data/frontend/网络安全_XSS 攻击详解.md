# 网络安全_XSS 攻击详解

## 一、核心定义：什么是 XSS？

### 【问题】
什么是 XSS？它的全称、本质和危害是什么？

### 【回答】
- **全称**：Cross-Site Scripting（跨站脚本攻击）。
- **本质**：攻击者想方设法向网页中注入恶意脚本代码（通常是 JavaScript，也包括 HTML），当用户访问该网页时，恶意代码会在用户的浏览器里被执行。
- **危害**：一旦执行，攻击者可以窃取用户 Cookie、伪造用户身份进行操作、篡改网页内容、甚至控制用户浏览器。

## 二、三大核心分类与原理

### 【问题】
XSS 的三种主要类型是什么？它们的注入方式和执行场景有何不同？

### 【回答】

#### 1. 存储型 XSS (Stored XSS)
- **别名**：持久型 XSS。
- **原理**：恶意代码被存储在服务器的数据库中。
- **流程**：攻击者将恶意代码提交到目标网站的数据库（如评论区、个人签名、帖子）→ 其他用户访问该页面时，服务器从数据库读取数据并将代码返回给用户浏览器 → 浏览器解析执行。
- **特点**：危害范围最大，所有访问该页面的用户都会中招，属于"一箭多雕"。

#### 2. 反射型 XSS (Reflected XSS)
- **别名**：非持久型 XSS。
- **原理**：恶意代码出现在 URL（网址）中，被服务器"反射"给浏览器。
- **流程**：攻击者构造包含恶意脚本的 URL → 诱导用户点击 → 服务器接收 URL 中的脚本并反射回页面（比如直接嵌入到 HTML 响应中）→ 浏览器执行。
- **特点**：需要用户配合点击特定链接才能触发，通常用于钓鱼攻击。

#### 3. DOM 型 XSS (DOM-based XSS)
- **原理**：漏洞产生于客户端（浏览器端）的 JavaScript 代码解析不当，与服务器端响应无关。
- **流程**：浏览器加载页面时，通过 JavaScript 直接从 URL 或 DOM 中读取恶意数据并写入到页面中执行。
- **特点**：请求完全不会到达服务器，属于"客户端漏洞"，很难被服务器端的防火墙拦截。

## 三、学习与防御视角（补充）

### 【问题】
作为前端开发者或安全学习者，如何防御 XSS 攻击？

### 【回答】
理解原理是为了更好防御。常见的防御手段包括：
1. 对用户输入进行严格的验证和过滤
2. 对输出内容进行 HTML 实体编码
3. 使用 Content Security Policy (CSP) 限制脚本执行
4. 设置 HttpOnly 标志保护 Cookie
5. 使用现代前端框架的自动转义机制
