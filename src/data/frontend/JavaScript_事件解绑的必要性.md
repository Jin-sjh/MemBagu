# 事件解绑的必要性

## 【问题】
为什么必须解绑事件？核心原因是什么？

## 【回答】
**核心原因**：事件绑定会建立 DOM 元素与回调函数的强引用，若仅删除 DOM 而不解绑事件，回调函数会持续持有 DOM / 组件的引用，导致内存无法被 GC 回收，最终引发内存泄漏。

## 【问题】
哪些场景必须解绑事件？

## 【回答】
- **组件卸载 / 页面销毁时**（Vue 的 `beforeUnmount`、React 的 `useEffect` 清理函数）

- **动态创建 / 销毁的 DOM 元素**

- **定时器、延时器、第三方实例绑定的事件**

## 【问题】
不同场景下如何正确解绑事件？

## 【回答】
- **原生**：`removeEventListener`（必须传入与绑定完全一致的回调函数，不能用匿名函数）

- **Vue**：`@event` 自动解绑，手动绑定需在 `beforeUnmount` 中移除

- **React**：`useEffect` 返回清理函数执行解绑

- **第三方库**：调用实例的 `off` / `removeEventListener` / `dispose` 方法

## 【问题】
为什么要做事件解绑？

## 【回答】
事件绑定会建立 DOM 与回调的强引用，若仅删除 DOM 不解绑，回调会持续持有 DOM / 组件的引用，导致对象无法被 GC 回收，引发内存泄漏。因此必须在组件卸载、页面销毁时，用 `removeEventListener` 等方式解绑事件。
