# 变量声明与作用域

## 【问题】
解释 `var`、`let` 和 `const` 的区别

## 【回答】

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函数作用域 | 块级作用域 | 块级作用域 |
| 变量提升 | 是 | 否（暂时性死区） | 否（暂时性死区） |
| 重复声明 | 允许 | 不允许 | 不允许 |
| 初始值 | 可不初始化 | 可不初始化 | 必须初始化 |
| 重新赋值 | 可以 | 可以 | 不可以 |

## 【问题】
下面的代码将输出什么到控制台？为什么？

```javascript
(function(){
var a = b = 3;
})();

console.log("a defined " + (typeof a !== 'undefined'));
console.log("b defined " + (typeof b !== 'undefined'));
```

## 【回答】

输出结果如下：

```
a defined false
b defined true
```

### 原因分析：

**1. 变量 `a` 输出 `false` 的原因：**
- `var a = b = 3;` 中的 `var a` 声明是在函数作用域内（IIFE 立即执行函数）
- 函数外部无法访问函数内部的局部变量 `a`
- 所以 `typeof a === 'undefined'`，结果为 `false`

**2. 变量 `b` 输出 `true` 的原因：**
- `b = 3` 没有使用 `var`、`let` 或 `const` 声明
- 这导致 `b` 成为**隐式全局变量**（在严格模式下会报错）
- 在浏览器环境中，`b` 会成为 `window` 对象的属性
- 所以函数外部可以访问到 `b`，`typeof b !== 'undefined'`，结果为 `true`

### 核心知识点：

`var a = b = 3;` 实际上等价于：

```javascript
b = 3;      // 隐式创建全局变量
var a = 3;  // 函数作用域内的局部变量
```

> ⚠️ **注意：** 这种写法是 JavaScript 中的常见陷阱，应该避免使用。正确的写法应该是：
> ```javascript
> var a = 3, b = 3;  // 两个都是局部变量
> // 或
> var b = 3;
> var a = b;         // 明确声明
> ```

## 【问题】
什么是作用域链？

## 【回答】

在 JS 中，变量查找是按照**当前作用域** → **外层作用域** → **全局作用域**的顺序一层层找，这个层级关系就是**作用域链**。作用域链是在**函数定义时就确定的**，和执行位置无关。

## 【问题】
谈一谈 JavaScript 作用域链。

## 【回答】

当执行一段 JavaScript 代码（全局代码或函数）时，JavaScript 引擎会为其创建一个作用域，又称为执行上下文（Execution Context）。

在页面加载后，会首先创建一个全局作用域，然后每执行一个函数，会建立一个对应的作用域，从而形成一条作用域链。每个作用域都有一条对应的作用域链，链头是全局作用域，链尾是当前函数作用域。

作用域链的作用是解析标识符。当创建（不是执行）函数时，会将 this、arguments、命名参数和该函数中的所有局部变量添加到当前作用域中。当 JavaScript 需要查找变量的时候（这个过程称为变量解析），它首先会从作用域链中的链尾（也就是当前作用域）查找是否有该变量属性。如果没有找到，就顺着作用域链（也就是全局作用域链）继续查找，直到查找到链头。如果仍未找到该变量，就认为这段代码的作用域链上不存在该变量，并抛出一个引用错误（Reference Error）的异常。

## 【问题】
下列代码将会输出什么？

```javascript
var foo = 1;
function demo() {
    console.log(foo);
    var foo = 2;
    console.log(foo);
}
demo()
```

## 【回答】

输出结果如下：

```
undefined
2
```

### 原因分析：

**1. 第一个 `console.log(foo)` 输出 `undefined` 的原因：**
- 由于变量声明前置（变量提升），函数内部的 `var foo = 2` 会被提升到函数作用域顶部
- 但只提升声明，不提升赋值
- 所以实际执行顺序相当于：

```javascript
var foo = 1;
function demo() {
    var foo;           // 变量声明被提升到顶部
    console.log(foo);  // undefined（此时 foo 已声明但未赋值）
    foo = 2;           // 赋值操作保留在原位置
    console.log(foo);  // 2
}
demo()
```

**2. 第二个 `console.log(foo)` 输出 `2` 的原因：**
- 此时 `foo = 2` 已经执行，变量被赋值为 2
- 所以输出 `2`

### 核心知识点：

- **函数声明与变量声明**会被 JavaScript 引擎隐式地提升到当前作用域的顶部
- 但是**只提升名称，不会提升赋值部分**
- 变量提升的优先级：函数声明 > 变量声明
- 使用 `let` 和 `const` 可以避免变量提升带来的问题（它们会形成暂时性死区）

## 【问题】
什么是变量作用域？

## 【回答】

变量的作用域是指变量有效的范围，与变量定义的位置密切相关，作用域是从空间这个角度来描述变量的，也可以理解为变量的可见性。在某个范围内变量是可见的，也就是说，变量是可用的。按照作用域的不同，变量可分为局部变量和全局变量。

在函数中使用 `var` 关键字显式声明的变量是局部变量；而没有用 `var` 关键字，用直接赋值的方式声明的变量是全局变量。

## 【问题】
创建的变量什么时候是局部变量？什么时候是全局变量？

## 【回答】
如果在函数里面用 var 声明，就是局部变量；如果在函数里面不用 var 声明，那就是全局变量；如果在函数外面用 var 定义，都是全局变量。

## 【问题】
说说你对作用域链的作用的理解。

## 【回答】
作用域链的作用是保证执行环境里，有权访问的变量和函数是有序的，作用域链中的变量只能向上访问，变量访问到全局（window）对象即终止，在作用域链中向下访问变量是不允许的。

## 【问题】
说出以下程序的运行结果。

```javascript
(function() {
    let a = this ? class b {} : class c {};
    console.log(typeof a, typeof b, typeof c)
})()
```

## 【回答】

结果为 `function undefined undefined`。

### 原因分析：

**1. `typeof a` 输出 `function` 的原因：**
- 这是一个立即执行函数表达式（IIFE），在严格模式或非严格模式下，`this` 的值取决于执行环境
- 在浏览器全局环境中，`this` 指向 `window` 对象（真值），所以执行 `class b {}`
- `class` 声明的类本质上是一个构造函数，所以 `typeof a` 返回 `"function"`

**2. `typeof b` 和 `typeof c` 输出 `undefined` 的原因：**
- 在定义函数/类变量时，**函数名称或类名称只能在函数体/类体中生效**
- `class b {}` 和 `class c {}` 中的 `b` 和 `c` 是类名，它们的作用域仅限于各自的类体内部
- 在类表达式外部，类名 `b` 和 `c` 是不可访问的
- 因此 `b` 和 `c` 在外部作用域中是未定义的，`typeof` 返回 `"undefined"`

### 核心知识点：

- **具名类表达式**（Named Class Expression）的类名只在类体内部可见
- 这与具名函数表达式类似：函数名只在函数体内部可用
- 类表达式的类名不会像函数声明那样在外部作用域创建绑定
