# TCP 三次握手与四次挥手

## 三次握手（建立连接）

### 【问题】
tcp 为什么需要三次握手建立连接？

### 【回答】
三次握手的原因：
- 三次握手才可以阻止重复历史连接的初始化（主要原因）
- 三次握手才可以同步双方的初始序列号
- 三次握手才可以避免资源浪费

### 【问题】
什么是 TCP 三次握手？

### 【回答】
**a. SYN：** 客户端发送 SYN 包请求建立连接。

**b. SYN-ACK：** 服务器收到 SYN 包后，回复 SYN-ACK 包。

**c. ACK：** 客户端收到 SYN-ACK 包后，回复 ACK 包，建立连接。

## 四次挥手（断开连接）

### 【问题】
什么是 TCP 四次挥手？

### 【回答】
**a. FIN：** 客户端发送 FIN 包，表示请求关闭连接。

**b. ACK：** 服务器收到 FIN 包后，回复 ACK 包。

**c. FIN：** 服务器准备好关闭连接后，发送 FIN 包。

**d. ACK：** 客户端收到服务器的 FIN 包后，回复 ACK 包，连接关闭。

### 【问题】
为什么 4 次握手中间两次不能变成一次？

### 【回答】
服务器收到客户端的 FIN 报文时，内核会马上回一个 ACK 应答报文，但是服务端应用程序可能还有数据要发送，所以并不能马上发送 FIN 报文，而是将发送 FIN 报文的控制权交给服务端应用程序：

- 如果服务端应用程序有数据要发送的话，就发完数据后，才调用关闭连接的函数；
- 如果服务端应用程序没有数据要发送的话，可以直接调用关闭连接的函数。

### 【问题】
挥手要四次、握手三次，原因总结？

### 【回答】
- **三次握手：** 建立连接时，双方只需同步确认收发能力，服务端可将应答与请求合并，一次完成，所以 3 次就够。
- **四次挥手：** 断开时 TCP 是全双工通信，双方需各自单独关闭发送通道，服务端应答和关闭不能合并，必须分两次，因此要 4 次。
