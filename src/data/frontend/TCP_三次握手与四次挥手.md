# TCP 三次握手与四次挥手

## 三次握手（建立连接）

### 【问题】
什么是 TCP 三次握手？

### 【回答】
**a. SYN：** 客户端发送 SYN 包请求建立连接。

**b. SYN-ACK：** 服务器收到 SYN 包后，回复 SYN-ACK 包。

**c. ACK：** 客户端收到 SYN-ACK 包后，回复 ACK 包，建立连接。

## 四次挥手（断开连接）

### 【问题】
什么是 TCP 四次挥手？

### 【回答】
**a. FIN：** 客户端发送 FIN 包，表示请求关闭连接。

**b. ACK：** 服务器收到 FIN 包后，回复 ACK 包。

**c. FIN：** 服务器准备好关闭连接后，发送 FIN 包。

**d. ACK：** 客户端收到服务器的 FIN 包后，回复 ACK 包，连接关闭。
