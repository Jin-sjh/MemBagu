# 前端路由权限控制

## 1. 问：前端路由权限控制的核心原则是什么？

### 答：

前端路由本身**无法真正防止用户通过修改 URL 绕过权限**。

前端路由守卫的作用，更多是**优化用户体验**，比如在用户无权限时自动跳转到 403 或登录页，减少无效请求，但它无法阻止恶意用户绕过前端校验，直接访问后端资源。

正确的权限控制方案，必须是**前后端配合**的：

1. **前端层面**：通过路由守卫在跳转前做一次权限校验，控制页面可见性
2. **后端层面**：核心的权限校验必须放在后端，所有接口都要校验用户 Token 和对应权限，敏感资源必须通过接口鉴权后再返回，不能直接暴露静态地址

只有这样，才能真正保证系统的权限安全。
