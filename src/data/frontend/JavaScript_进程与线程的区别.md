# 4. 进程与线程的区别 + 前端场景对应（必会）

【问题】
进程与线程的区别是什么？在前端场景中如何对应？

【回答】

## (1) 概念区分

| 维度 | 进程（Process） | 线程（Thread） |
|------|----------------|---------------|
| 资源拥有 | 拥有独立内存空间 | 共享进程内存 |
| 通信成本 | 高（IPC） | 低（共享内存） |
| 稳定性 | 一个崩溃不影响其他 | 一个崩溃可能拖垮整个进程 |
| 调度单位 | 进程 | 线程 |

## (2) 浏览器中的体现

| 前端场景 | 对应操作系统概念 |
|----------|----------------|
| Chrome 的每个 Tab 一个渲染进程 | OS 的独立 process |
| JS 主线程 | 渲染进程内的一个 OS thread |
| WebWorker | 渲染进程中新创建的 OS thread |
| Compositor / Raster Thread | 浏览器内部多线程渲染 |
| Node.js 的 Worker Threads | OS threads |

## (3) 为什么浏览器使用多进程？

- 单个页面崩溃不会影响其他页面
- 提升安全性（沙箱）
- 利用多核 CPU

## (4) 为什么 JS 本身不允许多线程访问 DOM？

避免出现：

- 写时竞争（race condition）
- 锁问题（mutex、deadlock）
- 内存一致性模型复杂化
