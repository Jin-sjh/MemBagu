# 4. 进程与线程的区别 + 前端场景对应（必会）

【问题】
进程与线程的区别是什么？在前端场景中如何对应？

【回答】

进程是操作系统分配资源的最小单位，线程是 CPU 调度的最小单位。进程之间资源独立、隔离、开销大，一个崩了不影响其他；线程共享进程资源，切换轻量，但一个线程崩溃会导致整个进程崩溃。

放到前端里，浏览器就是典型的多进程架构，每个标签页通常对应一个渲染进程。而在渲染进程内部，又分为 JS 引擎线程、GUI 渲染线程、事件线程、定时器线程、网络线程等，它们共同完成页面渲染和 JS 执行。

前端常说的 JS 是单线程，指的就是JS 引擎线程是单线程执行，所以会有事件循环、异步任务这些机制。
