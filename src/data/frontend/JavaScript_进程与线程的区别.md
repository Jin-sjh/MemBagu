# 进程与线程的区别（面试标准回答版）

【问题】
进程与线程的区别是什么？

【回答】
一句话总结：**进程是资源分配的最小单位，线程是 CPU 调度的最小单位；一个进程可包含多个线程，线程共享进程资源。**

## 核心区别

### 1. 定义与定位
- **进程**：程序的一次执行实例，拥有独立的地址空间。
- **线程**：进程内的执行单元，也叫轻量级进程。

### 2. 资源分配
- **进程**：有**独立**的内存空间、文件句柄、信号等资源。
- **线程**：**共享**所属进程的堆、全局变量、文件描述符，只拥有独立栈、程序计数器、寄存器。

### 3. 开销
- **进程**：创建、切换、销毁开销大。
- **线程**：开销小，切换速度快。

### 4. 通信方式
- **进程**：管道、消息队列、共享内存、信号量等。
- **线程**：直接读写全局变量即可，需配合同步机制（互斥锁、条件变量）。

### 5. 健壮性
- **进程**：一个进程崩溃，不影响其他进程。
- **线程**：一个线程崩溃，整个进程都会崩溃。

### 6. 调度
- **进程**：独立调度，但开销大。
- **线程**：CPU 调度的基本单位，切换高效。

## 通俗类比

- **进程 = 一个工厂**
- **线程 = 工厂里的工人**
- 工厂有独立场地、设备（独立资源）
- 工人共享工厂资源，但各干各的活（共享资源，独立执行）
- 工厂倒闭 = 进程挂了 → 所有工人都没了
- 一个工人出事 = 线程异常 → 整个工厂可能停工

## 前端场景对应

放到前端里，浏览器就是典型的多进程架构，每个标签页通常对应一个渲染进程。而在渲染进程内部，又分为 JS 引擎线程、GUI 渲染线程、事件线程、定时器线程、网络线程等，它们共同完成页面渲染和 JS 执行。

前端常说的 JS 是单线程，指的就是 JS 引擎线程是单线程执行，所以会有事件循环、异步任务这些机制。

【问题】
为什么进程崩溃不会对其他进程产生很大影响？

【回答】
主要是因为：

- **进程隔离性**：每个进程都有自己独立的内存空间，当一个进程崩溃时，其内存空间会被操作系统回收，不会影响其他进程的内存空间。这种进程间的隔离性保证了一个进程崩溃不会直接影响其他进程的执行。

- **进程独立性**：每个进程都是独立运行的，它们之间不会共享资源，如文件、网络连接等。因此，一个进程的崩溃通常不会对其他进程的资源产生影响。
