# HTTP/3 核心优势与工作原理

## 1. HTTP/3 基于 UDP 相比 TCP 有哪些核心优势？

【问题】HTTP/3 基于 UDP 相比 TCP 有哪些核心优势？

【回答】
- 彻底解决传输层队头阻塞，单个流丢包不影响其他流。
- 连接建立更快，首次 1 RTT，复用连接 0 RTT。
- 支持连接迁移，Wi-Fi/5G 切换 IP 不变断连。
- 强制 TLS 1.3，默认全加密，安全性更强。
- 协议在用户态实现，更新灵活，避免 TCP 协议僵化。

---

## 2. HTTP/3 解决了 HTTP/2 的哪些关键问题？

【问题】HTTP/3 解决了 HTTP/2 的哪些关键问题？

【回答】
- 解决 HTTP/2 因 TCP 导致的队头阻塞，丢包不再阻塞整个连接。
- 降低 HTTPS 建连延迟，从 2 RTT 优化到 1/0 RTT。
- 解决 TCP 四元组绑定问题，实现无缝网络切换。
- 改善弱网/高丢包环境下的性能，避免 HTTP/2 有时比 HTTP/1.1 更慢的问题。
- 内置加密，减少握手开销，提升安全性与部署一致性。

---

## 3. 为什么 HTTP/2 多路复用仍有性能瓶颈？

【问题】为什么 HTTP/2 多路复用仍有性能瓶颈？

【回答】
HTTP/2 多路复用只是在单个 TCP 连接上虚拟并行，TCP 丢包会导致整个连接等待重传，所有请求都被阻塞，这就是传输层队头阻塞。

---

## 4. 为什么 HTTP/3 用 UDP 还能保证可靠？

【问题】为什么 HTTP/3 用 UDP 还能保证可靠？

【回答】
HTTP/3 基于 QUIC，在 UDP 之上重新实现了可靠传输、流量控制、拥塞控制、重传、排序等 TCP 核心能力，同时避开了 TCP 的缺陷。

---

## 5. HTTP/3 的 0 RTT 是什么，有什么意义？

【问题】HTTP/3 的 0 RTT 是什么，有什么意义？

【回答】
0 RTT 指客户端复用之前的连接信息，第一个数据包就可以发送业务数据，无需额外握手，大幅降低首次请求延迟，提升页面打开速度。

---

## 6. 什么是连接迁移，HTTP/3 如何实现？

【问题】什么是连接迁移，HTTP/3 如何实现？

【回答】
连接迁移指网络切换（Wi-Fi 切蜂窝）时连接不断。

HTTP/3 使用 Connection ID 标识连接，不依赖 IP + 端口四元组，IP 变化也能继续传输。

---

## 7. 为什么说 HTTP/3 更适合移动/弱网环境？

【问题】为什么说 HTTP/3 更适合移动/弱网环境？

【回答】
- 丢包只影响单个流，不阻塞整体。
- 建连更快，重连成本低。
- 支持网络无缝切换。
- 拥塞控制更灵活，适配复杂网络。
