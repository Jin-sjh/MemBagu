# XSS 攻击与防护策略

【问题】
XSS 攻击是什么？

【回答】
XSS 是跨站脚本攻击，攻击者通过在 Web 页面中插入恶意脚本代码，然后诱使用户访问该页面，从而使得恶意脚本在用户浏览器中执行，从而盗取用户信息、会话信息等敏感数据，甚至控制用户账户。

【问题】
前端安全防护策略 (XSS/CSRF) 有哪些？

【回答】
## XSS 防护：

1. **输入验证**：对用户输入进行严格的过滤和转义。

2. **内容安全策略 (CSP)**：通过 CSP 限制可执行的脚本源。

## CSRF 防护：

1. **使用 Token**：在请求中加入 CSRF Token 进行校验。

2. **SameSite Cookies**：限制跨站请求带上 Cookie。
