# MVC 架构和 MVVM 架构

## 【问题】
说说你对 MVC 架构和 MVVM 架构的理解。

## 【回答】
在经典的 MVC 架构中，包含 3 个部分，即模型（Model）、视图（View）和控制器（Controller）。控制器可以访问视图，让其更新。控制器可以访问模型，更新数据。视图可以访问模型，获取数据渲染页面。

在 MVVM 架构中，包含 3 个部分，即模型（Model）、视图（View）和视图模型（ViewModel）。视图模型负责视图与模型之间的信息转换，通过数据双向绑定使视图与模型之间的数据得以传递。

例如代表性的框架 Angular，它通过数据绑定，将模型中的数据映射到视图中，通过事件监听器（event listener），将视图改变的数据存储在模型内。
