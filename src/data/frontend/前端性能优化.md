【问题】前端性能优化主要从哪些方面入手？（简洁版，1 分钟内说完）

【回答】
前端性能优化主要从网络、资源加载、渲染、代码、运行时几个方面入手：
1. 网络优化：减少请求数，合并资源，开启 Gzip/Brotli 压缩，使用 CDN，设置合理缓存。
2. 资源加载优化：图片懒加载、按需加载路由、使用 webp/avif 等现代图片格式、压缩图片。
3. 渲染优化：减少重排重绘，使用 CSS3 硬件加速，避免频繁操作 DOM。
4. 代码优化：减少冗余代码，Tree Shaking，避免内存泄漏，防抖节流。
5. 构建优化：代码分割、压缩混淆、第三方库按需引入。

【问题】请详细谈谈前端性能优化的完整方案（面试高分回答，条理清晰）

【回答】
我一般从网络层、资源层、渲染层、代码层、构建层、运行时六个维度做性能优化：

1. 网络层面
- 启用 HTTP/2、HTTP/3，多路复用减少请求阻塞
- 开启 Gzip / Brotli 压缩文本资源
- 使用 CDN 加速静态资源
- 配置强缓存与协商缓存（Cache-Control、ETag、Last-Modified）
- 减少 DNS 解析，使用 DNS Prefetch、Preconnect

2. 资源加载优化
- 图片优化：压缩、WebP/AVIF、懒加载、响应式图片 srcset
- 字体优化：font-display: swap，避免 FOIT/FOUT
- 脚本优化：script defer/async，关键 JS 内联
- 路由懒加载、组件异步加载，实现按需加载

3. 渲染优化（浏览器渲染流程）
- 减少 重排（Reflow）重绘（Repaint），主要从以下几方面做：
  - 读写分离：避免在循环或高频事件里读写 DOM 穿插，防止强制同步布局
  - 批量操作 DOM：使用 DocumentFragment 离线修改，只做一次插入
  - 动画优先用 transform 和 opacity：让浏览器生成独立合成层，用 GPU 加速，不触发重排重绘
  - 把频繁变化的元素脱离文档流：缩小重排影响范围
  - 避免频繁读取布局属性：减少 Layout 次数，提升渲染效率
- 批量 DOM 操作，使用文档片段 DocumentFragment
- 避免频繁读取 offsetTop/scrollTop 等布局属性
- 使用 transform、opacity 触发合成层，硬件加速
- 避免大面积使用 * 通配符、昂贵选择器

4. JavaScript 运行时优化
- 防抖（debounce）、节流（throttle）处理高频事件
- 避免内存泄漏（定时器未清除、全局变量滥用、闭包过度）
- 长任务拆分，使用 Web Worker 处理计算密集型逻辑
- 避免在 render / 更新中做复杂计算

5. 构建打包优化
- Tree Shaking 删除无用代码
- 代码分割（Code Splitting）
- 第三方库按需引入，替换轻量替代库
- 压缩 JS/CSS/HTML，移除 console、注释
- 分析打包体积（webpack-bundle-analyzer）

6. 体验与监控
- 使用骨架屏、loading 提升感知性能
- 预加载关键资源：preload、prefetch
- 接入 Lighthouse、Web Vitals 监控 FCP、LCP、CLS、FID 等指标

【问题】前端性能优化极简背诵版（30 秒）

【回答】
前端性能优化我主要做这几点：
网络压缩缓存、图片懒加载、资源按需加载、减少重排重绘、防抖节流、代码分割 Tree Shaking、使用 Lighthouse 监控核心指标持续优化。
