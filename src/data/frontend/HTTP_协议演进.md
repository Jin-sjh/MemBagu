# HTTP 协议演进

## 4. HTTPS 和 HTTP 的区别，HTTPS 是如何保证安全的

### 【问题】
HTTPS 和 HTTP 的区别，HTTPS 是如何保证安全的

### 【回答】

**HTTP：**
- 数据是明文传输的，容易被中间人（如黑客）窃听或篡改。

**HTTPS：**
- **加密**：使用 SSL/TLS 协议对传输的数据进行加密，确保数据在传输过程中的安全性。
- **身份验证**：使用 SSL/TLS 证书对服务器进行身份验证，防止假冒网站。
- **完整性校验**：通过数据摘要（哈希算法）来保证数据在传输过程中不被篡改。

## 3. HTTP/2 解决了什么问题，仍然解决不了的问题，HTTP/3+UDP 如何解决

### 【问题】
HTTP/2 解决了什么问题，仍然解决不了的问题，HTTP/3+UDP 如何解决

### 【回答】

**HTTP/2 解决的问题：**
- 解决了队头阻塞：通过多路复用，一个连接可以并发传输多个请求和响应。
- 头部压缩：减少了数据包头部的冗余数据，提高了带宽利用率。
- TCP 连接复用：减少了大量的 TCP 连接的建立和拆除开销。

**HTTP/2 无法解决的问题：**
- TCP 的固有问题：虽然解决了队头阻塞，但由于 TCP 仍然基于单个连接，存在 TCP 队头阻塞问题，特别是在高延迟和丢包场景下。
- 连接建立时的延迟：TCP 握手依然存在延迟。

**HTTP/3 + UDP：**
- UDP 没有 TCP 的队头阻塞问题，因此通过使用 QUIC 协议实现的 HTTP/3 能避免 TCP 连接中的队头阻塞。
- 多路复用：QUIC 协议允许多个流在一个 UDP 连接中并行传输，解决了 TCP 连接中的队头阻塞问题。
- 更低的连接延迟：QUIC 通过 0-RTT 连接重用，减少了连接建立的延迟。
- 更好的移动网络支持：由于 UDP 是无连接的，适合移动网络中频繁切换网络的情况。
