# 虚拟 DOM 原理

## 【问题】说说虚拟 DOM 的实现原理。

## 【回答】

我可以从**是什么、为什么、怎么做**三个方面来回答：

1. **虚拟 DOM 本质上就是用一个轻量的 JavaScript 对象，来模拟真实 DOM 树的结构**，包括标签名、属性、子节点等。它不依赖浏览器环境，操作成本远低于真实 DOM。

2. 它的核心流程分为三步：
   - 第一步，页面渲染时，先根据数据生成**虚拟 DOM 树**；
   - 第二步，当数据发生变化时，生成**新的虚拟 DOM 树**，并通过**diff 算法**和旧树做**同层对比**，找出最小差异；
   - 第三步，把这些差异**批量更新到真实 DOM**上，避免频繁操作 DOM 带来的重绘重排，从而提升性能。

3. diff 算法的核心思想是：
   - **只同层比较，不跨层级**；
   - **节点类型不同直接替换，不继续比较子节点**；
   - 配合**key**来复用节点、减少 DOM 创建和删除。

所以虚拟 DOM 的本质，就是**用 JS 计算成本换取 DOM 操作成本**，同时让框架实现**跨平台**和**声明式渲染**更方便。

---

## 极简版（15 秒回答，适合紧张时）

虚拟 DOM 就是用 JS 对象模拟真实 DOM 树。当数据变化时，通过 diff 算法对比新旧虚拟 DOM 树，找出最小差异后批量更新到真实 DOM，避免频繁操作 DOM 带来的性能问题。diff 算法只同层比较，节点类型不同直接替换，配合 key 复用节点。本质是用 JS 计算成本换取 DOM 操作成本。
