# 5. 页面卡顿的根本原因（高频解释题）

【问题】
页面卡顿的根本原因是什么？

【回答】
页面卡顿并不是 "JS 多，而是主线程被阻塞

## 主要原因：

### (1) JS 执行时间过长（最常见）

例如：

- 大量数据处理
- 复杂循环
- JSON 超大对象解析
- 加密算法、图像处理
- 解决方案：WebWorker/WASM（展望的时候才能说）

### (2) 频繁 DOM 修改导致 Layout / Reflow

构建 DOM 树、计算布局、合成绘制需要耗时。

### (3) 渲染层压力过大

- 大量动画
- 超大图像
- Canvas 连续高频重绘
- 纹理过大导致 GPU 内存爆掉

### (4) 内存泄漏

导致：

- 浏览器内存占用持续上升
- OS 触发 Swap/page fault → 卡顿非常明显
- 尤其是长时间运行的管理后台

### (5) 阻塞 I/O

如：

- localStorage 是同步 API
- 同步 XHR 造成主线程 freeze

这些必须避免。
