# 4. JS 为什么设计成单线程？

【问题】
JS 为什么设计成单线程？

【回答】
JS 设计为单线程，是由其运行环境和核心用途决定的，核心原因有 3 点：

1. **浏览器环境的需求**：JS 最初是为浏览器设计的，核心用途是操作 DOM。如果是多线程，多个线程同时操作同一个 DOM，会出现不可预期的渲染冲突（比如一个线程修改 DOM，另一个线程删除 DOM），单线程从根本上避免了这个问题；

2. **避免线程同步的复杂性**：多线程需要处理锁、同步、死锁等复杂问题，会大幅增加语言的复杂性和开发成本，不符合 JS「简单易用」的设计初衷；

3. **事件循环（Event Loop）的互补**：JS 通过「单线程 + 事件循环 + 任务队列」的模型，实现了异步非阻塞的效果，既保证了单线程的简单性，又解决了阻塞问题（比如网络请求、定时器不会阻塞主线程）。

【面试加分点】

- 单线程指的是**JS 引擎的主线程是单线程**，浏览器 / Node.js 有其他辅助线程（比如网络请求线程、定时器线程），这些线程会把回调任务放到任务队列中，由主线程执行；

- Web Worker 是 JS 的多线程方案，但 Worker 线程无法操作 DOM，仅能处理计算任务，本质上是「辅助线程」，不改变 JS 主线程的单线程特性；

- Node.js 的 JS 同样是单线程，通过 Libuv 实现异步 I/O，原理和浏览器一致。
