# 内存泄漏风险与优化

## 【问题】
内存泄漏的常见泄漏场景有哪些？

## 【回答】
1. **意外的全局变量**：
   ```javascript
   function foo() { bar = 1; }
   ```
   未声明直接赋值会创建全局变量，导致无法被垃圾回收

2. **闭包过度使用**：长期缓存大对象，闭包让外层作用域的变量无法被垃圾回收

3. **DOM 事件未移除**：组件卸载后事件监听仍然存在，导致 DOM 元素无法被回收

## 【问题】
内存泄漏的优化方案有哪些？

## 【回答】
1. **及时解除引用**：
   ```javascript
   timer = null;
   ```
   将不再使用的变量引用置为 null，帮助垃圾回收器识别

2. **使用 WeakMap 存储临时数据**：WeakMap 的键是弱引用，不会阻止垃圾回收

3. **组件卸载时清理事件监听**：在组件销毁前主动移除所有事件监听器，避免内存泄漏
