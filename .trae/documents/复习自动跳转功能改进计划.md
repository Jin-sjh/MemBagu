# 复习自动跳转功能改进计划

## 问题分析

当前在 `ReviewList.vue` 的 `handleAnswer` 函数中，自动跳转逻辑存在问题：

```javascript
function handleAnswer({ questionId, remembered }) {
  emit('review', { questionId, remembered })
  
  const idx = props.questions.findIndex(q => q.id === questionId)
  if (idx !== -1 && remembered) {  // 只有 remembered 为 true 时才跳转
    setTimeout(() => {
      if (currentQuestion.value && currentQuestion.value.id === questionId) {
        if (idx < props.questions.length - 1) {
          currentQuestion.value = props.questions[idx + 1]
        } else {
          closeReview()
        }
      }
    }, 300)
  }
}
```

**问题**：

1. 只有点击"记住了"才会自动跳转到下一题
2. 点击"没记住"后，用户需要手动关闭弹窗再选择下一个题目
3. 用户体验不连贯，复习效率低

## 解决方案

修改 `handleAnswer` 函数，使其在用户回答后（无论记住与否）都能自动跳转到下一题继续复习。

## 实现步骤

### 步骤 1：修改 ReviewList.vue 的 handleAnswer 函数

**文件**: `src/components/ReviewList.vue`

**修改内容**:

* 移除 `remembered` 条件判断，使无论回答结果如何都能自动跳转

* 保留 300ms 延迟，给用户一个短暂的视觉反馈时间

* 当复习完所有题目时，自动关闭弹窗

**修改后的代码**:

```javascript
function handleAnswer({ questionId, remembered }) {
  emit('review', { questionId, remembered })
  
  const idx = props.questions.findIndex(q => q.id === questionId)
  if (idx !== -1) {
    setTimeout(() => {
      if (currentQuestion.value && currentQuestion.value.id === questionId) {
        if (idx < props.questions.length - 1) {
          currentQuestion.value = props.questions[idx + 1]
        } else {
          closeReview()
        }
      }
    }, 300)
  }
}
```

## 预期效果

1. 用户点击"记住了"或"没记住"后，自动跳转到下一题
2. 复习完所有题目后，自动关闭弹窗
3. 复习流程更加流畅，提高学习效率
