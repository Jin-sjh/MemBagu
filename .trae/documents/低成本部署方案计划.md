# 八股整理项目低成本部署方案

## 项目现状分析

### 技术栈

* **前端**: Vue 3 + Vite (SPA 单页应用)

* **后端**: Express.js (音频生成服务，端口 3002)

* **数据存储**: localStorage (浏览器本地存储)

* **依赖服务**: edge-tts-nodejs (文本转语音)

### 当前架构特点

1. 前端是纯静态资源，构建后可直接部署
2. 后端服务用于音频生成和题库文件夹管理
3. 数据完全存储在浏览器 localStorage 中
4. 前后端通过 API 通信

### 步骤 4: 创建 Supabase 配置文件

新建文件 `src/utils/supabase.js`：

```javascript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(supabaseUrl, supabaseAnonKey)

export async function signIn(email, password) {
  const { data, error } = await supabase.auth.signInWithPassword({
    email,
    password
  })
  return { data, error }
}

export async function signUp(email, password) {
  const { data, error } = await supabase.auth.signUp({
    email,
    password
  })
  return { data, error }
}

export async function signOut() {
  const { error } = await supabase.auth.signOut()
  return { error }
}

export async function getCurrentUser() {
  const { data: { user } } = await supabase.auth.getUser()
  return user
}
```

### 步骤 5: 修改 storage.js 支持 Supabase

修改 `src/utils/storage.js`，添加云端同步函数：

```javascript
import { supabase, getCurrentUser } from './supabase.js'

// 原有 localStorage 函数保持不变，作为离线缓存

// 新增：云端同步函数
export async function syncProgressToCloud(libraryId, progress) {
  const user = await getCurrentUser()
  if (!user) return false
  
  const records = Object.entries(progress).map(([questionId, data]) => ({
    user_id: user.id,
    library_id: libraryId,
    question_id: questionId,
    status: data.status,
    next_review: data.nextReview,
    review_count: data.reviewCount,
    ease_factor: data.easeFactor,
    interval: data.interval
  }))
  
  const { error } = await supabase
    .from('progress')
    .upsert(records, { onConflict: 'user_id,library_id,question_id' })
  
  return !error
}

export async function loadProgressFromCloud(libraryId) {
  const user = await getCurrentUser()
  if (!user) return {}
  
  const { data, error } = await supabase
    .from('progress')
    .select('*')
    .eq('user_id', user.id)
    .eq('library_id', libraryId)
  
  if (error) return {}
  
  const progress = {}
  data.forEach(record => {
    progress[record.question_id] = {
      status: record.status,
      nextReview: record.next_review,
      reviewCount: record.review_count,
      easeFactor: record.ease_factor,
      interval: record.interval
    }
  })
  
  return progress
}
```

## 常见问题

### Q: 后端服务启动慢怎么办？
A: Render 免费层会在 15 分钟无访问后休眠，首次访问需要 30-60 秒唤醒。可以：
- 升级到付费计划（$7/月）获得不休眠服务
- 使用 UptimeRobot 等服务定时 ping 保持唤醒

### Q: Supabase 邮箱验证收不到？
A: 
- 检查垃圾邮件文件夹
- 在 Supabase 控制台 → Authentication → Users 手动确认用户

### Q: 如何自定义域名？
A:
- 在 Vercel 项目设置中添加自定义域名
- 需要自己购买域名并配置 DNS

### Q: 数据存储在哪里？
A:
- 用户账号和数据存储在 Supabase（云端）
- localStorage 作为离线缓存（本地）
- 音频文件临时存储在 Render 服务器
