# 八股分库功能实现计划

## 需求分析

当前项目是一个艾宾浩斯记忆辅助系统，所有八股题目都存放在 `src/data/` 目录下，属于单一的"前端八股库"。用户希望能够：

1. **创建多个题库**：如前端库、Python库、Java库等
2. **独立管理**：每个库有独立的分类、问题、学习进度
3. **自由切换**：用户可以在不同库之间切换

## 当前架构分析

### 数据层

* `src/data/*.md` - 存放所有 Markdown 格式的题目文件（目前都是前端相关）

* `src/utils/parser.js` - 解析 MD 文件，提取问题和答案

* 文件命名格式：`{分类}_{主题}.md`

### 存储层

* `src/utils/storage.js` - 使用 localStorage 存储进度和 UI 状态

* 存储键：`ebbinghaus_progress`、`ebbinghaus_ui_state`

### 核心逻辑

* `src/composables/useQuestions.js` - 问题数据管理

* `src/composables/useProgress.js` - 学习进度管理

* `src/composables/useEbbinghaus.js` - 艾宾浩斯算法

### 7. 注意事项

1. **数据隔离**：确保不同库的进度数据完全隔离
2. **性能优化**：切换库时避免重复加载
3. **错误处理**：库目录不存在或为空时的友好提示
4. **用户体验**：切换库时保持 UI 状态连续性
5. **向后兼容**：确保旧用户数据自动迁移到 frontend 库
